这个方案的工作模式

  • LLM 写起来 = 写 mermaid：5-20 行文本即可描述一张图，跟写 markdown 一样轻
  • 用户看 = 双击 .html：浏览器自动拉 mermaid CDN 渲染（首次联网，之后浏览器缓存）
  • git diff 友好：diff 看的是 mermaid 源码，不是二进制 PNG/SVG
  • self-contained：单文件，可以邮件 / 复制走

集成进 x-req

x-req 在写 README.md 的同时，多吐一个 diagram.html 到 task 目录。两个文件分工：

  • README.md = 文字事实源（LLM 维护）
  • diagram.html = 可视化只读视图（mermaid 源码 + HTML 壳）

用户审需求：双击 diagram.html 看大局；要改细节：去 README.md 改文字 + 同步改 diagram.html 的 mermaid 块。
