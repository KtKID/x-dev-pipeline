# 正确性 CR 报告模板

输出 CR 报告时使用本模板。模板保留 `审查结论` 和 `问题详情` 两个章节，供 `x-fix/references/cr-fix-mode.md` 解析。

---

## 模板骨架

```markdown
# Correctness Review 报告

> Report ID: YYYYMMDD-HHmmss
> Mode: known-issue | module-review
> Scope: task | diff | module | file | PR
> Task ID: <如有>
> Review 日期：YYYY-MM-DD
> 审查范围：<文件 / 模块 / diff / PR>

## 调查对象

| 字段 | 内容 |
|------|------|
| 模式 | 已知问题 / 模块正确性 review |
| 用户现象 | <用户报告的问题；模块 review 写 N/A> |
| 期望行为 | <用户期望 / spec 期望 / 模块契约> |
| 实际行为 | <日志、测试、代码路径或 diff 证据> |
| 原始 spec 来源 | `<path>` / 用户消息 / PR 描述 / 测试契约 |

## 修改文件 / 审查范围

| 文件 | 角色 | 语言 | 说明 |
|------|------|------|------|
| `<path>` | implementation / test / config / docs | `<language>` | <说明> |

---

## 贝叶斯根因调查

| H | 先验 | 证据 E | 影响 | 更新后置信度 | 下一步 |
|---|------|--------|------|--------------|--------|
| H1: <候选根因> | 低 / 中 / 高 | <证据> | 支持 / 削弱 / 中性 | 低 / 中 / 高 / 已确认 | <下一步> |

### 已排除假设

| H | 排除证据 |
|---|----------|
| Hn: <已排除根因> | <排除证据> |

---

## Spec 对照

| 项 | 原始 spec / 契约 | 当前实现 / 行为 | 分类 | 证据 |
|----|------------------|----------------|------|------|
| <检查项> | <spec / 契约> | <当前实现 / 行为> | <分类> | <证据> |

分类取值：原始 spec 不一致 / 实现过程偏移 / spec 缺口 / 环境数据问题 / 证据不足 / 符合 spec。

---

| ❌ / ✅ | <来源> | <检查项> | `<path>:<line>` / - | <置信度> | <根因分类> | <描述> |

| 状态 | 来源 | 检查项 | 文件:位置 | 置信度 | 根因分类 | 描述 |
|------|------|--------|-----------|--------|----------|------|
| ⚠️ / ✅ | <来源> | <检查项> | `<path>:<line>` / - | <置信度> | <根因分类> | <描述> |

### P2：正确性证据缺口

| 状态 | 来源 | 检查项 | 文件:位置 | 置信度 | 根因分类 | 描述 |
|------|------|--------|-----------|--------|----------|------|
| ⚠️ / ✅ | <来源> | <检查项> | `<path>:<line>` / - | <置信度> | <根因分类> | <描述> |
```

## 填充示例：已知问题调查

```markdown
# Correctness Review 报告

> Report ID: 20260629-153000
> Mode: known-issue
> Scope: task
> Task ID: login-session-fix
> Review 日期：2026-06-29
> 审查范围：`src/auth/login.ts`、`src/auth/session.ts`

## 调查对象

| 字段 | 内容 |
|------|------|
| 模式 | 已知问题 |
| 用户现象 | 密码输错后再次登录会进入半登录状态 |
| 期望行为 | 登录失败后清理本次 session 临时状态 |
| 实际行为 | 失败分支直接 return，session 临时字段保留 |
| 原始 spec 来源 | `dev-pipeline/tasks/login-session-fix/README.md` |

## 修改文件 / 审查范围

| 文件 | 角色 | 语言 | 说明 |
|------|------|------|------|
| `src/auth/login.ts` | implementation | TypeScript | 登录失败分支 |
| `src/auth/session.ts` | implementation | TypeScript | session 状态管理 |

---

## 贝叶斯根因调查

| H | 先验 | 证据 E | 影响 | 更新后置信度 | 下一步 |
|---|------|--------|------|--------------|--------|
| H1: token 过期 | 中 | 同 token 在另一个环境可用 | 削弱 | 低 | 已排除 |
| H2: 失败分支未清理 session | 中 | `login.ts:47` 失败分支直接 return | 支持 | 高 | 查调用方 cleanup |
| H2: 失败分支未清理 session | 高 | 调用链未发现 finally cleanup | 支持 | 已确认 | 写入 P0 |

### 已排除假设

| H | 排除证据 |
|---|----------|
| H1: token 过期 | 刷新 token 后仍失败，同 token 在另一个环境可用 |

---

## Spec 对照

| 项 | 原始 spec / 契约 | 当前实现 / 行为 | 分类 | 证据 |
|----|------------------|----------------|------|------|
| 登录失败状态清理 | 登录失败后清理本次 session 临时状态 | 失败分支直接 return | 实现过程偏移 | `login.ts:47` |

---

## 审查结论

### P0：已确认正确性错误

| 状态 | 来源 | 检查项 | 文件:位置 | 置信度 | 根因分类 | 描述 |
|------|------|--------|-----------|--------|----------|------|
| ❌ | 用户现象 | 状态清理 | `login.ts:47` | 已确认 | 实现过程偏移 | 登录失败分支未清理 session 临时状态 |

### P1：生产正确性风险

| 状态 | 来源 | 检查项 | 文件:位置 | 置信度 | 根因分类 | 描述 |
|------|------|--------|-----------|--------|----------|------|
| ✅ | 代码路径 | 错误冒泡 | - | 低风险 | 符合 spec | 上层错误中间件会转成统一业务错误 |

### P2：正确性证据缺口

| 状态 | 来源 | 检查项 | 文件:位置 | 置信度 | 根因分类 | 描述 |
|------|------|--------|-----------|--------|----------|------|
| ⚠️ | 测试契约 | 回归测试缺口 | `tests/login.test.ts` | 中 | spec 缺口 | 缺少“失败后再次登录”的回归测试 |

---

## 问题详情

### B1: 登录失败分支未清理 session 临时状态
**来源**: 用户现象 + 代码路径
**文件**: `src/auth/login.ts`
**位置**: 第 47 行
**严重程度**: P0
**根因分类**: 实现过程偏移
**置信度**: 已确认

**问题描述**：
密码校验失败时直接返回，未清理本次登录尝试写入 session 的临时字段。

**贝叶斯更新**：
- H：失败分支未清理 session
- 先验：中
- 支持证据：`login.ts:47` 失败分支直接 return；调用链未发现 finally cleanup
- 削弱证据：未发现上游统一清理逻辑
- 反证检查：检查 login 调用方、session helper、错误中间件
- 更新后置信度：已确认

**Spec 对照**：
- 原始 spec / 契约：README 要求登录失败后清理临时状态
- 当前实现 / 行为：失败分支直接 return
- 分类：实现过程偏移

**影响**：
用户一次失败登录会污染下一次登录尝试，可能出现半登录状态。

**验证方式**：
增加回归测试：先输入错误密码，再输入正确密码，断言 session 中无失败尝试残留字段。

**修复建议**：
在失败分支返回前清理 session 临时字段，或把登录流程包进 `try/finally`。

---

## 汇总

| 等级 | 数量 |
|------|------|
| P0 | 1 |
| P1 | 0 |
| P2 | 1 |

## 最终结论

- **不建议合并**，有 1 个 P0 正确性错误。
- 根因结论：
  - 登录失败分支未清理 session，属于实现过程偏移。
- 必须修复项：
  - B1 登录失败状态清理。
- 需补证或确认项：
  - 增加失败后再次登录的回归测试。
```

```markdown
### P2：正确性证据缺口
```
